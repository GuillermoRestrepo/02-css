MARGIN PROBLEMS

existen dos problemas principales al tiliza margenes en css.

Colapsado De Margenes

Conocido como (margin collapsing), es un problema que tienen los margenes verticales.

Al haber dos elementos, uno encima del otro, los margenes verticales no se suman, estos se super ponen. Ejemplo:

Código: Como Se Ve: Como Deberia De Verse:

Código HTML:


<body>
    <div Class="eb1"></div>
    <div Class="eb2"></div>
</body>

Código CSS:


.eb1 {
    background-color: orange;
    width: 150px;
    height: 150px;
    margin: 20px
    border: 1px solid black;
}

Como se puede notar, el margen inferior del primer elemento, y el margen superior del segundo elemento, se superponen, haciendo q solo haya una separación de 20 píxeles, en lugar de 40 como debería de ser.

PARA SOLUCIONAR ESTO, SE SUELE DAR UNICAMENTE MARGENES INFERIORES, EVITANDO ASÍ LA SUPERPOSICION DE MARGENES.

Margen De Elemento Hijo

Los margenes de un elemento hijo, puede desbordarse del elemento padre, y puede afectar a la posicion de este ultimo.

Código:

Como aparece por defecto:

El margen de 10px del contenedor hijo, se desborda del contenedor padre en su eje vertical.

Solución #1:

Controlar el contenido desbordado del contenedor padre (overflow: hidden;)

Solución #2:

Dar un margen interno diminuto al contenedor padre (padding: 0.1px;)

Solución #3:

Dar al contenedor padre un borde superior diminuto y del mismo color (border-top: 0.1px solid orange;).

Código HTML:


<body>
    <div class="ElementoPadre">
        <h2 class="ElementoHijo">Encabezado</h2>
    </div>
</body>

Código CSS:


.ElementoPadre {
    background-color: orange;
    width: 250px;
    height: 50px;
}

.ElementoHijo {
    margin: 10px;
}
